Full Views Listing¶

Listed below is the complete list of views for the JustHealth web application, broken down by their general function.

Each method will show its:

  • URL
  • Primary function
  • Method type (POST or GET)
  • POST / GET parameters
  • Possible return values

User Login¶

justHealthServer.views.login()[source]¶

Gets the username from login in order to authenticate session, log the user in and redirects to index function

URL:/login
HTTP_METHOD:POST, GET
Parameters:username – Checks username is valid, posts to authenticate() API
Return values:
  • Redirects to appropriate home page for user
  • Fields not filled in/Input not valid

Home¶

justHealthServer.views.index(*args, **kwargs)[source]¶

Sends user to a dashboard according to account type if session is active, it shows their profile and connections. A patient will be able to see their notifications, prescriptions and appointments. A carer will be able to see their notifications, the patients that they are connected to (with prescriptions and appointments) and their own appointments

URL:/
Parameters:accounttype – Checks the type of account user has registered with
Return values:
dashboard.html
  • Sends the user to a home page designed specifically for a patient’s needs. Shows them the basic information they need in one place, and allows them to navigate to other pages easily.
dashboardCarer.html
  • Sends the user to a home page designed specifically for a carer’s needs, simplifying the administrative work involved with care.

Gets the user’s profile picture from the JustHealth images folder on the server

URL:/images/<filename>

Return values:
  • Profile Picture

Search¶

justHealthServer.views.search(*args, **kwargs)[source]¶

Search to find a different user, potentially so a connection can then be requested

search.html

NHS Search¶

justHealthServer.views.searchNHS(*args, **kwargs)[source]¶

The POST request is no longer needed as this is done with Javascript

URL:/nhsSearch
HTTP_METHOD:POST, GET
Parameters:searchTerm – Provides the text that is sent to the NHS website to be searched for
Return values:
searchNHSDirect.html
  • After sending the search term to the NHS website, it opens the page in a new tab showing the results.
  • The page remains on the search form once submitted.

Deactivate¶

justHealthServer.views.deactivate(*args, **kwargs)[source]¶

Handles account deactivation form

URL:/deactivate
HTTP_METHOD:POST, GET
Parameters:username – Finds account by username

“Your account has been deleted”
  • User account is deleted and all information is removed from the database.
“Your account has been deactivated”
  • User account is deactivated and their information remains in the database.
Deactivation failed

Forgot Password¶

justHealthServer.views.forgotPassword()[source]¶

Finds the user’s account from the email address it was registered with

URL:/forgotPassword
HTTP_METHOD:POST, GET
Parameters:email – Finds user account from valid email
Return values:
“An account with this email address does not exist.”
  • Message shown if email address is not found in the database.
“An email has been sent to you containing a link, which will allow you to reset your password.”
  • Message informs the user that they can successfully reset their password.

Password Reset¶

justHealthServer.views.resetPasswordRedirect()[source]¶

Follows unique link in email to allow user to reset password, only after forgotPassword form is submitted.

URL:/resetpassword
HTTP_METHOD:POST, GET
Parameters:resetPassword() – Allows user to reset password if "True"
Return values:
“Your password has been reset, please check your email and click the link to verify.”
  • Successful password reset, user has to click the link in their email to prove the reset was them.
Failed
  • Fields not filled in/Input not valid

Delete Prescription¶

justHealthServer.views.deletePrescription_view(*args, **kwargs)[source]¶

Informs Carer that the prescription has been deleted from the selected patient

URL:

/deletePrescription
Parameters:
  • username – Patient’s medication is stored in the database associated with their username
  • prescriptionid – Uses the prescription ID to delete a specific prescription
Return values:
Success/Deleted

Failed
  • myPatients.html, shows the prescription still exists
