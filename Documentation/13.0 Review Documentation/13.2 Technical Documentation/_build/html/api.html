Full API Listing¶

Listed below is the complete list of possible API functionality provided by JustHealth, broken down by their general function.

All API calls require valid HTTP Basic Authentication using a registered username and encrypted password unless otherwise noted.

Accounts¶

justHealthServer.api.addAndroidEventId(*args, **kwargs)[source]¶

Allows an android event id of a calendar event to be stored.

Parameters:request.form (dict.) – POST request containing [dbid](id of event) and [androidid](the android event id).
Returns:str – Success message
justHealthServer.api.addCorrespondence(details)[source]¶

Adds a note for a patient.

Parameters:details (dict.) – Dictionary containing [carer], [patient], [notes] (note content), [title].
Returns:str – True or False for success.
justHealthServer.api.addDeactivate(reason)[source]¶

Allows a new add deactivate reasons to be added to the database.

Parameters:reason (str.) – The reason to add.
Returns:str – True or False for success.
justHealthServer.api.addReminders(username, now)[source]¶

Add all new reminders to the Reminder table.

Parameters:
  • username (str.) – The username to add reminders for.
  • now (datetime.) – The current datetime.
justHealthServer.api.authenticate()[source]¶

Authenticates a username and password and returns the result of the authentication check

justHealthServer.api.cancelRequest(details)[source]¶

Cancels the user request to connect before completion

justHealthServer.api.changePasswordAPI(details)[source]¶

Allows a user to change their password. POST [username, oldpassword, newpassword, confirmnewpassword]

justHealthServer.api.checkMissedPrescriptions(username, currentDate)[source]¶

Checks to see if a patient has missed any prescriptions and creates notifications if so.

Parameters:
  • username (str.) – The username to check for.
  • currentDate (datetime.) – The current datetime.
justHealthServer.api.checkStockLevel(prescription, count)[source]¶

Returns the stock level of a prescription after taking. Creates a notification is appropriate.

Parameters:
  • prescription (int.) – The id of the prescription.
  • count (int.) – The amount just taken.
justHealthServer.api.completeConnection(details)[source]¶

Verify a code on input to allow the completion of an attempted connection. POST[ username, requestor, codeattempt]

justHealthServer.api.createConnection(details)[source]¶

Creates an initial connection between two users. POST [username, target]

justHealthServer.api.createNotificationRecord(user, notificationType, relatedObject)[source]¶

Creates a notification on the JustHealth platform.

Parameters:
  • user (str.) – The user the notification belongs to.
  • notificationType (str.) – The type of notification to be created
  • relatedObject (int.) – The resource a notification pertains to, if any.
Returns:

str – ‘True’ if successful, else ‘False’
justHealthServer.api.createTakePrescriptionInstances(username, currentDateTime)[source]¶

Creates all TakePrescription instances for any prescription that needs to be taken today.

Parameters:
  • username (str.) – The username to check for.
  • currentDateTime (datetime.) – The current datetime.
justHealthServer.api.deactivateAccount(details)[source]¶

justHealthServer.api.decryptPassword(cipherText)[source]¶

Decrypts the users password and returns it so that we are able to authenticate them

justHealthServer.api.deleteAccount(username)[source]¶

Deletes a user’s account.

Parameters:username (str.) – The username who’s account should be deleted.
Returns:str – ‘Deleted’ if successful, else ‘Failed’.
justHealthServer.api.deleteNote(noteid)[source]¶

Parameters:noteid (int.) – The id of the note to delete.

Dismisses a notification to hide from the user’s immediate view.

Parameters:notificationid (int.) – The id of the notification to dismiss.
Returns:str – ‘True’ if successful, ‘False’ if not.
